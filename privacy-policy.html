1. Overview

Poker Points is a FigJam plugin that helps agile teams assign story point estimates to stickies directly on the canvas. This policy explains what data the plugin accesses, what it stores, and what it does not do.

2. Data We Do Not Collect

Poker Points does not collect, transmit, or store:

  • Your name, email address, or any account information
  • The content of your FigJam boards, stickies, or files
  • Usage data, analytics, or telemetry
  • Device identifiers or IP addresses
  • Any information identifying your organisation or team

3. What the Plugin Reads from Your Canvas

When running, the plugin reads only:

  • The position and size of selected sticky nodes (x, y, width, height) — to place the estimate badge correctly
  • The text value of existing estimate badges — to compute totals and averages in the summary bar

This data is processed entirely within the Figma plugin sandbox and is never transmitted anywhere.

4. Local Storage

The plugin uses Figma's built-in clientStorage API to remember any custom estimation values you configure (e.g. a modified Fibonacci sequence). This data:

  • Never leaves Figma's environment
  • Is scoped exclusively to this plugin
  • Contains only the estimation scale values you have typed in (e.g. "1, 2, 3, 5, 8")
  • Can be cleared at any time by resetting your custom values

5. Network Access

Poker Points makes no network requests. The plugin manifest explicitly declares an empty allowed-domains list ("allowedDomains": []), meaning it is technically incapable of contacting any external server.

6. Third-Party Services

The plugin includes no third-party libraries, analytics SDKs, advertising code, or external scripts of any kind. All code runs locally within Figma.

7. Children's Privacy

Since the plugin collects no personal data from anyone, it inherently complies with COPPA and similar regulations regarding minors.

8. GDPR & CCPA

Because no personal data is collected or processed, Poker Points has no data controller or processor obligations under GDPR (EU) or CCPA (California). There is no personal data to access, correct, or delete.

9. Changes to This Policy

If a future version of the plugin introduces any data collection (e.g. optional cloud sync), this policy will be updated before release and the change will be clearly communicated in the plugin's release notes.
